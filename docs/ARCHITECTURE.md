# Architecture & Technical Design

## Overview

This document describes the technical architecture of the AI-Powered Resume Screening & Skill Gap Analyzer.

## Processing Pipeline

### 1. PDF Text Extraction (`utils/pdf_parser.py`)
- Uses `pypdf.PdfReader` to iterate through all pages
- Extracts raw text and performs basic line-level cleanup
- Handles corrupted or empty PDFs gracefully

### 2. NLP Preprocessing (`utils/text_preprocessing.py`)
- **Text Cleaning**: Removes URLs, emails, non-ASCII characters, and special punctuation
- **Tokenization**: NLTK `word_tokenize` with fallback to whitespace splitting
- **Stopword Removal**: Filters English stopwords using NLTK corpus
- **Lemmatization**: Reduces tokens to base forms using `WordNetLemmatizer`
- **NLTK Auto-initialization**: Downloads required resources on first run with timeout protection

### 3. Skill & Experience Extraction (`utils/skill_extractor.py`)
- **Skill Database**: 17 predefined tech skills (Python, Java, SQL, ML, DL, NLP, etc.)
- **Matching**: Case-insensitive regex with word boundaries
- **Gap Analysis**: Set operations to identify matching/missing skills
- **Recommendations**: Curated learning resources mapped to each missing skill

### 4. Hybrid Score Calculation (`utils/matcher.py`)
- **TF-IDF Cosine Similarity** (50% weight): Semantic overlap between resume and JD
- **Skill Overlap Ratio** (40% weight): Fraction of JD skills present in resume
- **Experience Fit** (10% weight): Linear scaling based on experience match
- **Final Score**: Weighted combination normalized to 0–100 scale

### 5. ML Classification (`utils/ml_model.py`)
- **Models**: Logistic Regression, Random Forest (100 trees), XGBoost (100 rounds)
- **Features**: `[similarity_score, matched_skills_count, missing_skills_count, experience_years]`
- **Training**: 80/20 stratified split on 300 synthetic candidate records
- **Evaluation**: Accuracy, Precision, Recall, F1 Score, ROC-AUC
- **Selection**: Best model chosen by F1 Score

### 6. SHAP Explainability (`utils/ml_model.py`)
- **TreeExplainer**: Used for Random Forest and XGBoost
- **Generic Explainer**: Used for Logistic Regression (with training data background)
- **Output**: Waterfall plot showing per-feature SHAP values for a single prediction

## Data Flow

```
PDF Upload → Text Extraction → NLP Cleanup → Feature Engineering → ML Prediction → SHAP Explanation
                                    ↓                  ↓
                              Skill Extraction    Hybrid Scoring
                                    ↓                  ↓
                              Gap Analysis      Leaderboard Ranking
```

## Synthetic Dataset

Generated by `data_generator.py`:
- **300 records** with correlated features
- Similarity scores: Normal distribution (μ=62, σ=12)
- Matched skills: Correlated with similarity + Poisson noise
- Shortlist labels: Sigmoid probability with 7% label noise
